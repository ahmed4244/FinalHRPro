# Senior Security Engineer (the first promotion)

Rung 1 on the [[cyber-security-career-ladder]], the target at **year five**. You
stop working handed-down tickets and start **owning a security domain** — detection
& response, or the security of a whole product — end to end.

## The scope
You run real incidents, threat-model live systems, and set the security bar for your
area ([[the-scope-ladder-task-to-org]]). The org trusts your judgment on security
calls within your domain.

## What it takes
- **Technical:** deep in one domain (appsec, detection & response, cloud security);
  can threat-model a real system and drive the fixes; hands-on offensive *and*
  defensive skill.
- **Leadership:** mentor junior analysts, run security reviews, communicate risk to
  engineers in terms they act on.
- **Experience:** ~5 years.

## Certifications
- **CompTIA Security+** (the baseline), then a hands-on GIAC — **GCIH** (incident
  handling) or **GCIA** (intrusion analysis); **OSCP** for the offensive track.

## Courses & reading
- ***The Web Application Hacker's Handbook*** and the **OWASP Top 10** for appsec depth.
- A **blue-team / detection-engineering** course (e.g. SANS SEC-series) for the defensive track.

## Projects that prove it
- **Own a security domain end-to-end** — e.g. build and run detection for a product.
- **Lead an incident** from detection through remediation and the postmortem.
- **Threat-model a major service** and drive the fixes to done.
- **Mentor a junior analyst** and run the security reviews for your area.

## How the decision is made
Show you already operate at Senior scope
([[promotion-is-operating-at-the-next-level]]): incidents you led, risks you drove
down measurably, and peers who trust your security judgment.
