# Staff / Principal Security Engineer (the second promotion)

Rung 2 on the [[cyber-security-career-ladder]], the target at **year ten**. You move
from owning one domain to **setting the security direction several teams build
against** — architecture, standards, and a program that raises the whole org.

## The scope
You are the security architect for a domain or the org: you design controls that are
hard to misuse, set the standards teams follow, and multiply other engineers' output
([[the-scope-ladder-task-to-org]]). Gated on org-wide impact, not tenure.

## What it takes
- **Technical:** security architecture across systems; connect controls to business
  risk and make the tradeoffs explicit; deep expertise plus broad judgment.
- **Leadership:** influence across teams without authority; sponsor senior engineers;
  represent security in design forums and hiring.
- **Experience:** ~9–12+ years.

## Certifications
- **CISSP** — the management/architecture standard, the cornerstone here.
- **CCSP** (cloud security) or an advanced offensive cert (**OSCE/OSEP**) for depth.

## Courses & reading
- ***Threat Modeling: Designing for Security*** (Adam Shostack) and a **security
  architecture** framework (SABSA) for the architect track.

## Projects that prove it
- **Design the security architecture** for a platform used by multiple teams.
- **Establish an org-wide security program** — a secure SDLC or a zero-trust rollout.
- **Own a multi-quarter security strategy** for a domain and drive it to adoption.
- **Grow senior security engineers** and drive the org's security standards.

## How the decision is made
Sustained org-wide impact ([[promotion-is-operating-at-the-next-level]]): an
architecture teams build on, a program that measurably lowered risk, and Staff+
sponsors who attest to it.
